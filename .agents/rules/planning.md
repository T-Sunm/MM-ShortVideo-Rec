# Planning Guidelines

## Task Summary
Before starting any implementation, summarize the task in 3-5 clear bullet points.

## Minimal Plan
Propose a plan that:
- Fixes or implements the requested feature with the least amount of change.
- Maintains architectural boundaries (logic vs UI).
- Reuses existing components.

## File Selection
List every file that will be modified. Avoid modifying more than 6 files for a single task.

## Verification Strategy
State exactly how you will verify that the change works (e.g., which pytest command to run, which page to check in Streamlit).
